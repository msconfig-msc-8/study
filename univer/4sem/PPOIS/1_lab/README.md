# Модель агентства недвижимости (CLI)

Программная система для лабораторной работы №1 по варианту 28.
Предметная область: посредничество в сделках с недвижимостью.

## Назначение проекта
Проект моделирует работу агентства недвижимости в консольном формате.
Пользователь может просматривать объекты, искать недвижимость по
критериям, организовывать просмотры, оценивать рыночную стоимость,
оформлять сделку и просматривать связанные документы.

## Что реализовано
- сущности предметной области: агентство, агент, клиент, объект
  недвижимости, документ, сделка, рынок недвижимости;
- операции поиска объектов недвижимости;
- операция проведения просмотров;
- операция оценки рыночной стоимости объекта;
- операция оформления документов;
- операция заключения сделки;
- обработка ошибочных ситуаций через исключения;
- интерфейс командной строки;
- unit-тесты.

## Основные сущности
- `Agency` — центральная сущность проекта, хранит агентов, клиентов,
  объекты недвижимости, сделки, документы и текущее состояние рынка.
- `Agent` — агент по недвижимости, который работает с клиентами и
  участвует в просмотрах и сделках.
- `Client` — клиент агентства с заданным бюджетом.
- `Property` — объект недвижимости с адресом, площадью, ценой и
  признаком доступности.
- `Market` — модель состояния рынка недвижимости через коэффициент
  изменения цен.
- `Document` — документ по сделке, который можно оформить и подписать.
- `Deal` — завершенная сделка, связанная с клиентом, агентом, объектом
  недвижимости и документом.

## Сервисы
- `SearchService` — ищет доступные объекты по максимальной цене и
  минимальной площади.
- `ViewingService` — организует просмотр объекта для выбранного клиента
  и агента.
- `ValuationService` — рассчитывает рыночную стоимость объекта с учетом
  состояния рынка.
- `DocumentService` — создает и подписывает документы.
- `DealService` — выполняет полный сценарий сделки: проверяет бюджет,
  продает объект, формирует документ и завершает сделку.

## Структура проекта
- `models/` — сущности предметной области.
- `services/` — бизнес-логика и операции.
- `tests/` — тесты моделей.
- `test_services/` — тесты сервисов.
- `docs/architectura.md` — краткое архитектурное описание.
- `main.py` — точка входа CLI-приложения.

## Что умеет CLI
- показать все объекты недвижимости;
- выполнить поиск по максимальной цене и минимальной площади;
- организовать просмотр по выбранным `id` клиента, агента и объекта;
- оценить стоимость объекта с учетом коэффициента рынка;
- провести сделку по выбранным `id`;
- показать список сделок;
- показать список оформленных документов.

## Сценарий работы сделки
1. Пользователь выбирает клиента, агента и объект недвижимости.
2. Система проверяет, доступен ли объект для продажи.
3. Система проверяет, хватает ли клиенту бюджета.
4. Из бюджета клиента списывается стоимость объекта.
5. Объект помечается как проданный.
6. Формируется и подписывается документ.
7. Создается завершенная сделка.
8. Сделка и документ сохраняются в агентстве.

## Обработка ошибок
В проекте используется механизм исключений Python.
Примеры ошибочных ситуаций:
- отрицательный бюджет клиента;
- нулевая или отрицательная цена объекта;
- попытка продать уже проданный объект;
- недостаточный бюджет клиента при сделке;
- повторное подписание документа;
- попытка организовать просмотр недоступного объекта.

## Основные решения
- `Agency` хранит все ключевые сущности и текущее состояние рынка;
- `DealService` создает завершенную сделку `Deal` и оформляет связанный
  `Document`;
- `ValuationService` использует сущность `Market`, а не случайный
  коэффициент;
- поиск работает только по доступным объектам, исключая уже проданные;
- консольный интерфейс позволяет выполнять основные операции предметной
  области через ввод пользователя.
